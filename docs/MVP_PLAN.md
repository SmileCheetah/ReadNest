# ReadNest MVP 구현 계획

## 1단계: 프로젝트 기본 구조

- NestJS 백엔드 프로젝트 생성
- Prisma 설치 및 MySQL 연결
- 환경 변수 구성
- 공통 PrismaModule 구성
- 기본 Health Check API 추가

완료 기준:

- 서버가 실행된다.
- Prisma가 MySQL에 연결된다.
- 첫 마이그레이션을 실행할 수 있다.

## 2단계: 인증

- 회원가입
- 로그인
- JWT 발급
- JWT Guard
- 내 정보 조회

완료 기준:

- 사용자는 계정을 만들고 로그인할 수 있다.
- 보호된 API는 로그인한 사용자만 호출할 수 있다.

## 3단계: 저장글 CRUD

- URL 저장
- URL 정규화
- 사용자별 중복 저장 방지
- 저장글 목록 조회
- 저장글 상세 조회
- 저장글 삭제

완료 기준:

- 같은 사용자가 같은 URL을 중복 저장할 수 없다.
- 저장글은 사용자별로 분리된다.
- 목록은 날짜 기준으로 조회할 수 있다.

## 4단계: 읽음 상태

- 읽음 처리
- 안 읽음 처리
- 나중에 다시 보기 처리

완료 기준:

- 저장글마다 `UNREAD`, `READ`, `READ_LATER` 중 하나의 상태를 가진다.
- 사용자는 상태를 변경할 수 있다.

## 5단계: 요약 큐

- Redis, BullMQ 설정
- 요약 작업 등록
- Worker 구성
- 요약 상태 관리
- 실패 시 재시도

완료 기준:

- URL 저장 후 요약 작업이 큐에 등록된다.
- Worker가 작업을 처리하고 결과를 DB에 저장한다.
- 실패한 작업을 재시도할 수 있다.

## 6단계: AI 요약

- 원문 추출 흐름 정의
- AI 요약 프롬프트 작성
- 제목, 짧은 요약, 핵심 포인트, 태그 저장
- 맥락 부족 상태 처리

완료 기준:

- 저장글 상세 화면에 필요한 요약 데이터가 생성된다.
- 요약 실패와 맥락 부족 상태가 구분된다.

## 7단계: 프론트 MVP

- 홈 화면
- 아카이브 화면
- 상세 화면
- 설정 화면
- 로그인 화면

완료 기준:

- 사용자는 로그인 후 URL을 저장하고 목록에서 확인할 수 있다.
- 저장글 상세에서 요약, 핵심 포인트, 태그, 원문 링크를 볼 수 있다.
- 읽음 상태를 변경할 수 있다.

## 8단계: 연속 글 감지

- 본문에서 `1/2`, `2/2`, `1 2`, `2 2` 계열 패턴 감지
- ThreadGroup, ThreadPart 저장
- 일부 파트만 있는 경우 맥락 부족 표시
- 모든 파트가 모이면 통합 요약 생성

완료 기준:

- 연속 글 일부만 저장된 경우 사용자에게 맥락 부족 가능성을 알린다.
- 관련 파트를 같은 묶음으로 연결할 수 있다.

## 9단계: 공유 저장

- Android 또는 PWA 공유 대상 검토
- 공유된 URL을 저장 API로 연결
- 저장 완료 피드백 제공

완료 기준:

- 사용자가 외부 앱의 공유 버튼에서 ReadNest로 URL을 전달할 수 있다.

## 10단계: 요약 활용 액션

- 상세 화면에서 요약 내용을 클립보드에 복사
- 상세 화면에서 요약 내용을 OS 공유 시트로 공유
- 공유 텍스트 포맷 정의
  - 제목
  - AI 요약
  - 주요 포인트
  - 원본 링크

완료 기준:

- 사용자는 요약된 내용을 다른 앱에 바로 붙여넣거나 공유할 수 있다.

## 첫 구현 추천 범위

처음에는 백엔드부터 시작합니다.

1. NestJS 프로젝트 생성
2. Prisma와 MySQL 연결
3. User, SavedArticle 모델 작성
4. 회원가입, 로그인 구현
5. URL 저장과 목록 조회 구현

이 범위가 끝나면 ReadNest의 가장 작은 뼈대가 생깁니다.
