# 프로젝트 기획서

# 프로젝트명

ReadNest

# 프로젝트 한 줄 설명

SNS와 웹에서 발견한 유용한 글을 공유 한 번으로 저장하고 AI가 요약하여 날짜별 아카이브로 정리해주는 개인 지식 큐레이션 서비스

# 프로젝트 목적

사람들은 Threads 블로그 뉴스 GitHub 같은 여러 플랫폼에서 유용한 글을 발견하지만 대부분 나중에 봐야지 하고 저장만 해둔 뒤 다시 읽지 않는다

ReadNest는 이런 문제를 해결하기 위해 사용자가 링크를 직접 복사하거나 앱에 붙여넣지 않아도 공유 버튼만으로 글을 저장할 수 있게 한다

저장된 글은 AI가 자동으로 요약하고 핵심 내용을 정리하며 사용자는 오늘 이번 주 이번 달처럼 기간별로 정리된 요약 아카이브를 확인할 수 있다

이 프로젝트의 핵심은 저장 자체가 아니라 저장한 글을 다시 읽기 쉬운 형태로 바꾸는 것이다

# 해결하려는 문제

사용자는 좋은 글을 발견해도 따로 정리하기 귀찮다

SNS에서 저장한 글은 시간이 지나면 다시 찾기 어렵다

좋아요나 저장 기능은 많지만 저장한 내용을 요약하거나 주제별로 정리해주지는 않는다

GPT에 직접 링크를 넣어 요약할 수도 있지만 앱을 열고 링크를 복사하고 붙여넣는 과정이 번거롭다

긴 글이나 연속 글은 원문을 다시 읽기 전까지 핵심을 파악하기 어렵다

# 핵심 사용자

SNS에서 정보성 글을 자주 저장하는 사람

Threads에서 개발 공부 취업 자기계발 트렌드 글을 많이 보는 사람

블로그 뉴스 GitHub 문서 링크를 저장해두고 다시 읽지 못하는 사람

공부 자료를 모으지만 정리하지 못하는 학생

AI를 활용해 학습 자료를 정리하고 싶은 개발자 지망생

# 핵심 사용 흐름

사용자는 Threads나 웹에서 좋은 글을 발견한다

공유 버튼을 누른다

공유 대상에서 ReadNest를 선택한다

ReadNest는 링크를 자동으로 저장한다

서버는 저장된 링크를 분석하고 요약 작업을 시작한다

AI는 제목 요약 핵심 포인트 태그를 생성한다

사용자는 앱에서 오늘 저장한 글 이번 주 저장한 글 월별 아카이브를 확인한다

글을 누르면 상세 요약과 원문 링크를 볼 수 있다

# MVP 핵심 기능

# 공유 저장

사용자는 외부 앱에서 공유 버튼을 통해 ReadNest에 링크를 저장할 수 있어야 한다

초기 개발 단계에서는 링크 입력으로 테스트할 수 있지만 최종 MVP 목표는 공유 기반 저장이다

저장된 링크는 사용자 계정과 연결되어야 한다

중복 저장된 링크는 새로 만들지 않고 기존 글을 보여주거나 중복 저장 상태를 알려준다

# 저장글 목록

저장된 글은 날짜 기준으로 정리한다

기본 구분은 오늘 이번 주 지난주 이번 달 월별 아카이브로 한다

각 글은 제목 요약 상태 출처 저장 시간 태그 읽음 상태를 보여준다

요약이 완료되지 않은 글은 요약 중 상태로 표시한다

# AI 요약

저장된 글은 AI 요약 대상이 된다

요약 결과에는 제목 짧은 요약 핵심 포인트 태그가 포함되어야 한다

요약은 비동기 작업으로 처리한다

요약 실패 시 사용자는 재시도할 수 있어야 한다

# 상세 화면

사용자는 저장글을 눌러 상세 화면으로 이동할 수 있다

상세 화면에는 제목 출처 저장 날짜 요약 핵심 포인트 태그 원문 링크가 표시된다

사용자는 원문 링크를 눌러 원래 글로 이동할 수 있다

사용자는 읽음 안 읽음 나중에 다시 보기 상태를 변경할 수 있다

# 읽음 상태 관리

각 저장글은 읽음 상태를 가진다

기본 상태는 안 읽음이다

사용자는 글을 읽음 처리할 수 있다

중요한 글은 나중에 다시 보기로 표시할 수 있다

# 연속 글 감지

Threads에는 1 2 또는 2 2처럼 여러 글로 나뉜 연속 글이 존재할 수 있다

서비스는 본문에서 연속 글 패턴을 감지해야 한다

일부 파트만 저장된 경우 맥락이 부족할 수 있음을 표시한다

사용자가 다른 파트를 추가 저장하면 같은 묶음으로 연결할 수 있어야 한다

연속 글이 모두 저장되면 통합 요약을 생성할 수 있다

# 요약 상태 관리

저장글은 처리 상태를 가진다

상태는 저장됨 요약 중 요약 완료 요약 실패 맥락 부족으로 나눈다

사용자는 현재 글이 어떤 상태인지 목록과 상세 화면에서 확인할 수 있어야 한다

# 제외할 기능

MVP에서는 좋아요 자동 수집을 구현하지 않는다

Threads의 모든 댓글 자동 수집은 구현하지 않는다

Chrome 확장 프로그램은 초기 범위에서 제외한다

iOS Share Extension은 초기 범위에서 제외한다

관심사 그래프는 초기 범위에서 제외한다

주간 월간 리포트는 초기 범위에서 제외한다

알림 기능은 초기 범위에서 제외한다

YouTube 영상 요약은 초기 범위에서 제외한다

GitHub Star 연동은 초기 범위에서 제외한다

# 화면 구성

# 홈 화면

홈 화면은 사용자가 가장 최근 저장한 글을 확인하는 공간이다

오늘 저장한 글을 먼저 보여준다

요약 중인 글을 별도로 보여준다

안 읽은 글을 확인할 수 있어야 한다

# 아카이브 화면

아카이브 화면은 기간별 저장글을 확인하는 공간이다

오늘 이번 주 지난주 이번 달 월별 아카이브 순서로 구성한다

사용자는 특정 기간을 선택해 저장글 목록을 볼 수 있다

# 상세 화면

상세 화면은 하나의 저장글을 깊게 확인하는 공간이다

요약 핵심 포인트 태그 원문 링크 읽음 상태를 제공한다

연속 글이 감지된 경우 해당 상태를 함께 보여준다

# 설정 화면

설정 화면은 계정 정보와 기본 설정을 관리하는 공간이다

초기에는 계정 정보 로그아웃 요약 언어 설정 정도만 제공한다

# 데이터 모델

# 사용자

사용자 정보는 로그인과 저장글 소유권 구분에 사용한다

필요한 정보는 이메일 닉네임 생성일이다

# 저장글

저장글은 사용자가 공유한 하나의 콘텐츠를 의미한다

필요한 정보는 사용자 식별자 출처 원문 링크 제목 작성자 원문 텍스트 요약 핵심 포인트 태그 처리 상태 읽음 상태 저장일 생성일이다

# 연속 글 묶음

연속 글 묶음은 1 2 2 2처럼 나뉜 글을 하나의 콘텐츠로 연결하기 위해 사용한다

필요한 정보는 사용자 식별자 묶음 제목 묶음 상태 생성일이다

# 연속 글 조각

연속 글 조각은 하나의 연속 글 안에 포함된 개별 파트를 의미한다

필요한 정보는 묶음 식별자 저장글 식별자 파트 번호 전체 파트 수 본문 링크 생성일이다

# 인증

로그인 기능이 필요하다

로그아웃은 프론트에서 토큰 삭제 방식으로 처리할 수 있다

저장글 중복 확인 기능이 필요하다

읽음 처리 기능이 필요하다

연속 글 패턴 감지 기능이 필요하다

연속 글 통합 요약 기능이 필요하다

# 기술 스택

프론트엔드는 Next js 또는 React 기반으로 구현한다

백엔드는 NestJS와 TypeScript로 구현한다

데이터베이스는 PostgreSQL 또는 MySQL을 사용한다

ORM은 Prisma를 사용한다

비동기 작업은 Redis와 BullMQ를 사용한다

AI 요약은 OpenAI 또는 Gemini API를 사용한다

인증은 JWT 기반으로 구현한다

배포는 Docker를 고려한다

# 백엔드 설계 방향

Controller는 요청과 응답을 담당한다

Service는 핵심 비즈니스 로직을 담당한다

Repository 또는 Prisma Service는 데이터 접근을 담당한다

DTO를 사용해 요청 데이터를 검증한다

예외 처리는 NestJS의 Exception 구조를 따른다

요약 작업은 요청과 동시에 처리하지 않고 큐에 넣어 비동기로 처리한다

사용자는 저장 직후 요약 중 상태를 확인할 수 있어야 한다

# AI 요약 정책

AI는 원문을 기반으로 제목 짧은 요약 핵심 포인트 태그를 생성한다

원문만으로 맥락이 부족하면 맥락 부족 상태를 표시한다

연속 글의 일부만 저장된 경우 전체 내용을 단정하지 않는다

요약은 짧고 명확하게 작성한다

태그는 최대 3개에서 5개 정도로 제한한다

광고성 문구나 불필요한 감탄 표현은 제거한다

# 디자인 방향

전체 디자인은 Notion처럼 차분하고 정리된 느낌을 목표로 한다

SNS 피드처럼 자극적인 디자인은 피한다

흰 배경과 연한 회색 구분선을 중심으로 구성한다

카드는 텍스트 중심으로 만든다

태그는 작은 pill 형태로 표시한다

상태는 저장됨 요약 중 요약 완료 요약 실패 맥락 부족처럼 명확하게 보여준다

핵심 화면은 홈 아카이브 상세 설정으로 단순하게 유지한다

# 사용자 경험 원칙

저장 과정은 최대한 짧아야 한다

사용자는 앱을 열고 링크를 붙여넣지 않아야 한다

공유 한 번으로 저장이 끝나야 한다

요약이 바로 완료되지 않아도 사용자는 현재 상태를 알 수 있어야 한다

목록 화면에서는 빠르게 훑을 수 있어야 한다

상세 화면에서는 원문을 다시 읽지 않아도 핵심을 파악할 수 있어야 한다

원문 링크는 항상 접근 가능해야 한다

프로젝트 기본 구조를 만든다

NestJS 서버를 생성한다

Prisma와 데이터베이스를 연결한다

사용자 인증 기능을 만든다

# 2단계

저장글 CRUD를 만든다

URL 저장 기능을 만든다

저장글 목록 조회와 상세 조회를 만든다

읽음 상태 변경 기능을 만든다

# 3단계

AI 요약 기능을 만든다

요약 요청을 큐에 넣는다

Worker가 요약을 처리한다

요약 결과를 저장한다

요약 상태를 화면에 보여준다

# 4단계

프론트 화면을 만든다

홈 화면을 만든다

상세 화면을 만든다

설정 화면을 만든다

# 7단계

중복 저장 처리를 개선한다

요약 실패 재시도를 추가한다

# 성공 기준

사용자는 Threads에서 공유 버튼으로 글을 저장할 수 있어야 한다

AI 요약이 자동으로 생성되어야 한다

사용자는 요약과 원문 링크를 상세 화면에서 볼 수 있어야 한다

연속 글 패턴을 감지할 수 있어야 한다

전체 흐름이 저장 요약 정리 다시 읽기로 자연스럽게 이어져야 한다

# 포트폴리오 소개 문장

ReadNest는 SNS와 웹에서 발견한 유용한 글을 공유 한 번으로 저장하고 AI가 요약하여 날짜별 아카이브로 정리해주는 개인 지식 큐레이션 서비스입니다

사용자가 직접 링크를 복사하거나 붙여넣지 않아도 공유 기능을 통해 글을 저장할 수 있으며 저장된 콘텐츠는 비동기 작업 큐를 통해 요약됩니다

Threads의 연속 글 패턴을 감지하여 여러 파트로 나뉜 글을 하나의 콘텐츠로 묶을 수 있도록 설계했습니다

이 프로젝트를 통해 NestJS 기반 API 설계 비동기 작업 처리 AI 요약 데이터 모델링 사용자 중심 UX 설계를 경험하는 것을 목표로 합니다

너는 숙련된 풀스택 개발자이자 코드 리뷰어다

나는 ReadNest라는 프로젝트를 만들고 있다

ReadNest는 SNS와 웹에서 발견한 유용한 글을 공유 한 번으로 저장하고 AI가 요약하여 날짜별 아카이브로 정리해주는 개인 지식 큐레이션 서비스다

기술 스택은 NestJS TypeScript Prisma PostgreSQL Redis BullMQ Next js를 기준으로 한다

우선순위는 기능을 많이 만드는 것이 아니라 MVP 흐름을 안정적으로 완성하는 것이다

핵심 흐름은 공유 URL 저장 요약 작업 등록 AI 요약 저장 날짜별 목록 조회 상세 조회 읽음 상태 변경이다

코드를 작성할 때는 다음 기준을 지켜라

기능을 작은 단위로 나누어 구현한다

Controller Service DTO Module 구조를 명확히 유지한다

비즈니스 로직은 Service에 둔다

요청 검증은 DTO로 처리한다

데이터 접근은 Prisma를 사용한다

요약 처리는 BullMQ 기반 비동기 작업으로 분리한다

에러 처리는 명확하게 한다

처음부터 과도한 추상화는 하지 않는다

각 단계가 끝나면 어떤 파일을 만들었고 어떤 역할을 하는지 설명한다

내가 학습할 수 있도록 코드 동작 흐름을 함께 설명한다

먼저 프로젝트 폴더 구조와 1차 MVP 구현 계획을 제안해줘
