# ReadNest 남은 작업 정리

이 문서는 현재 코드 기준으로 남은 작업을 우선순위대로 정리한 체크리스트입니다.

## 현재 기준 상태

2026-07-14 확인 기준:

- API 빌드 통과: `readnest-api`에서 `npm run build`
- API 테스트 통과: `readnest-api`에서 `npm test -- --runInBand`
- 모바일 타입 체크 통과: `readnest-mobile`에서 `npm run typecheck`
- 로컬 Homebrew MySQL / Redis 기준 API 통합 흐름 통과: 회원가입, 저장글 생성, 요약 큐 처리, 읽음 상태 변경, 삭제
- Expo Metro dev server 기동 확인: `exp://192.168.0.4:8081`, `http://localhost:8081`
- 모바일 API 주소 환경변수 분리 완료: `EXPO_PUBLIC_API_BASE_URL`
- 모바일 아카이브 검색/기간/읽음 필터 API query 연결 완료
- 백엔드 핵심 기능 구현됨: 인증, 저장글 CRUD, 읽음 상태, 요약 큐, Gemini 요약, Threads 원문 추출, 연속 글 감지 1차
- 모바일 핵심 화면 구현됨: 로그인, 홈, 아카이브, 상세, 설정, 저장, 삭제, 읽음 상태 변경, 요약 재시도, 복사, 공유

## 1순위: 모바일 실제 기기 QA

목표: 개발 PC 안에서만 되는 상태를 벗어나 실제 사용 환경에서 확인합니다.

- [ ] iOS 시뮬레이터에서 로그인/저장/요약 확인
- [ ] Android 에뮬레이터 또는 실제 Android 기기에서 확인
- [ ] 실제 iPhone 또는 개발 빌드에서 확인
- [ ] 앱 재실행 후 토큰 유지 확인
- [ ] 네트워크 끊김 상태의 에러 표시 확인
- [ ] API 서버가 꺼져 있을 때 에러 표시 확인
- [x] 모바일에서 접근 가능한 API 주소 설정 방식 정리

주의할 점:

- 모바일 API 주소는 `EXPO_PUBLIC_API_BASE_URL`로 바꿀 수 있습니다.
- 실제 기기에서는 `localhost`가 개발 PC가 아니라 기기 자신을 가리킬 수 있으므로 Mac의 LAN IP를 사용합니다.
- 환경변수 변경 후에는 `npm run start -- --clear`로 Metro 캐시를 비우는 것이 좋습니다.

완료 기준:

- 실제 기기에서 API 서버 주소 문제 없이 저장 흐름이 동작합니다.
- 사용자가 실패 원인을 이해할 수 있는 에러 메시지를 봅니다.
